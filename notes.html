KEY CONCEPTS

Lesson 1: Basics of HTML

WHAT IS THE WEB

Major pieces of the web include: me, my computer/browser, the internet, and servers. Each component interacts to result in the experience we've become accustomed to: I open a browser, type the URL "google.com" and HTTP sends a request to the appropriate server. That server then responds by delivering the requested HTML file back to the requester.

HTML

HTML - Hyper Text Markup Language - is a file-type availabe for use on the web - the most prevalent file-type, in fact. It's the heart-beat of the web. HTML can be text content, markup, references to other docs, or links to other pages.

HTML MARKUP

"Markup" is how we edit HTML. It's made up of and uses "tags".

<tag> "content" </tag> The above represents an element.

EXAMPLES OF TAGS

<a> = anchor (inline tag) <p> = paragraph (block tag) <div> = block tag that allows for styling <span> = inline tag (similar to div, but inline) <b> = bold (inline tag) <strong> = similar to bold <form> = block tag

LINKING

"href" tag is for linking (means hypertext reference)
example:

<a href="www.espn.com"> SPORTS NEWS! </a> NOTE: don't forget the quotation marks!
EXAMPLE IN ACTION: SPORTS NEWS!

Images are linked using "src" tag. See w3schools.com for more when needed.

INLINE VS. BLOCK TAGS

Inline tags and block tags act differently
Inline tags: act simply
Block tags: have height and width. Literally form a box around the text/etc

HTML DOCUMENT STRUCTURE

HTML docs begin with ""!doctype" (simply "HTML" in most cases)
then head closed by "/head" - includes metadata, css, and page header/title info
then body closed by "/body" - all of the main contents of the document/page

Example:

<!DOCTYPE html> <html> <head> <title>Document title</title> </head> <body> <h1>Page title</h1> <p> Lorem ipsum dolor sit amet, consectetur adipisicing elit </p> </body> </html>

NOTE: Code in a way that is easily read by others. Document well. Use tabs.

Forget something? Google it or go to w3schools.com (lots of great documentation)

Lesson 2: HTML AND PAGE STRUCTURE

PAGE STRUCTURE

HTML and CSS are both "languages". HTML controls the "structure" of a web page while CSS controls the "style" of a page - how it looks. "DOM" (Document Object Model) is the tree-like structure of a page, as viewed through the 'developer tools' offered within Chrome, for example.

BOXIFYING

First, it's worth noting that 'boxifying' is a real word - def would've lost that bet, but now I'll win Scrabble.

Basically, everything we see on a webpage is comprised of a group/series of boxes. Within these boxes the 'magic' is happening - HTML and CSS are working together to generate a visual output we like.

Developers 'boxify' design copy in order to simplify HTML development and administration.
